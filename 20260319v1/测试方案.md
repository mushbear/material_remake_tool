- **方案3、4**: 使用 qwen3.5-plus AI分析素材并生成，以F06结尾

## 4. 剪辑要求

### 4.1 场景使用规则
- 每个原始素材的每个场景只能使用一次
- F06-悬念结尾/付费卡点可以复用

### 4.2 场景打标
为每个素材场景标记在原片中的位置（集数+该集中第几个场景）：
- **匹配规则**（权重从高到低）：
  1. plot_summary 相似度（最高）
  2. 出场人物相似度（次高）
  3. 时间长度相似度（最低）
- **匹配逻辑**：取相似度最高的原片场景
- **集数来源**：从 video_name 获取

### 4.3 未匹配场景处理
- 未匹配且分类为F06：正常使用
- 未匹配且分类不为F06：只能放在剪辑后素材的第一个场景使用，或不使用

### 4.4 连续场景合并
- 如果连续场景的 narrative_function_tag 相同，合并为一个场景
- 合并后的时长为所有场景时长之和

### 4.5 时长要求
- F01-强开局/吸睛钩子：不限制
- F06-悬念结尾/付费卡点：不限制
- 其他场景：必须超过15秒（合并场景以合并后时长为准）

### 4.6 最后场景处理
- 如果选择的场景是素材的最后一个场景且不是F06，去掉最后1秒

### 4.7 顺序验证
- 场景顺序需与原片一致
- 如不一致，尝试替换场景或放弃该方案
- 找不到适合的场景则放弃该方案输出

## 5. 实现步骤

### Step 1: 数据准备
1. 复制 material_list.csv 到输出目录
2. 加载所有素材JSON和原片JSON到内存
3. 构建原片场景索引（集数 → 场景列表）

### Step 2: 场景匹配模块
```
SceneMatcher:
- compute_similarity()  # 计算综合相似度（摘要+人物+时长）
- match_scene()         # 单场景匹配，取最高相似度
- match_all_scenes()    # 批量匹配所有素材场景
```

### Step 3: 场景打标
```
SceneTagger:
- tag_scene()           # 打标单个场景（集数+场景序号）
- handle_unmatched()    # 处理未匹配场景
```

### Step 4: 连续场景合并
```
SceneMerger:
- merge_consecutive()   # 合并连续相同打标场景
```

### Step 5: 方案生成
```
SchemeGenerator:
- generate_fixed()      # 生成固定方案(1,2)
- generate_ai()         # 使用qwen3.5-plus AI生成方案(3,4)
```

### Step 6: 方案验证
```
SchemeValidator:
- validate_duration()   # 验证时长（非F01/F06必须>15秒）
- validate_order()      # 验证原片顺序一致性
- adjust_or_reject()    # 调整或放弃方案
```

### Step 7: 视频剪辑
```
VideoComposer:
- cut_segment()         # 使用ffmpeg剪辑单个片段
- merge_video()         # 合并视频片段
```
