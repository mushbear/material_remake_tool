# 素材组合工具需求文档

## 项目概述
本工具用于根据指定的剧情标签组合，从现有素材中提取对应场景，并重新组合成新的素材文件，最终产出N个新素材（去除BGM）。

## 模块二：场景信息提取

### 功能描述
根据模块一输出的素材ID列表（或测试给定的素材ID列表），提取每个素材对应的场景列表及场景相关属性。

### 输入
- 素材ID列表（测试时通过参数传入）
- 示例：`["1326922", "1327758", "1330863"]`

### 数据来源
- **JSON分析文件目录**：项目目录下（通过配置文件传入路径）
- **参考格式**：`sample.json`
- **素材下载信息**：`material_list.csv`

### 场景属性格式（根据sample.json）
每个场景包含以下信息：
- `segment_id`: 场景编号（对应scene_number）
- `start_time`: 场景起始时间（秒，整数）
- `end_time`: 场景结束时间（秒，整数）
- `duration`: 场景时长（秒）
- `narrative_function_tag`: 剧情功能标签（如：F01-强开局/吸睛钩子、F02-背景速递/设定交代、F03-极限施压/受辱等）
- `plot_summary`: 场景剧情摘要
- `main_location`: 主要场景位置
- `emotion_trope_tags`: 情绪标签列表

### 输出格式

## 模块三：场景组合方案生成

### 功能描述
根据给定的 `narrative_function_tag` 组合规则，从模块二提取的场景库中，匹配对应标签的场景，生成N个素材组合方案。

### 输入
- **标签组合规则**：指定场景的标签序列
  - 示例：`F01-强开局/吸睛钩子 + F02-背景速递/设定交代 + F03-极限施压/受辱`
- **模块二的场景数据**：所有可用场景及其标签
- **方案数量**：需要生成的组合方案数量N

### 组合逻辑
1. 根据标签组合规则，从场景库中筛选匹配每个标签的场景
2. 为每个位置（第一场景、第二场景...）随机或按策略选择一个匹配场景
3. 生成N个不同的组合方案

### 输出格式
每个方案包含N个场景，格式如下：

```json
{
  "scheme_id": 1,
  "scenes": [
    {
      "scene_index": 1,
      "material_id": "1326922",
      "scene_number": 1,
      "scene_start_time": "00:00:00",
      "scene_end_time": "00:00:15",
      "narrative_function_tag": "F01-强开局/吸睛钩子"
    },
    {
      "scene_index": 2,
      "material_id": "1327758",
      "scene_number": 3,
      "scene_start_time": "00:00:45",
      "scene_end_time": "00:01:00",
      "narrative_function_tag": "F02-背景速递/设定交代"
    },
    {
      "scene_index": 3,
      "material_id": "1330863",
      "scene_number": 2,
      "scene_start_time": "00:00:20",
      "scene_end_time": "00:00:35",
      "narrative_function_tag": "F03-极限施压/受辱"
    }
  ],
  "total_duration": 45
}
```

### 输出文件
- 保存为JSON格式：`scene_combination_schemes.json`

## 模块四：素材下载与组合

### 功能描述
根据模块三生成的组合方案，下载对应素材片段，去除BGM，拼接成新素材。

### 输入
- 模块三输出的组合方案：`scene_combination_schemes.json`
- 素材元数据文件：`material_list.csv`

### 数据来源
- 素材下载信息：`material_list.csv`
  - 字段：`id`, `material_name`, `video_url`

### 处理流程
1. **读取组合方案**
   - 解析 `scene_combination_schemes.json`

2. **下载原始素材**
   - 根据 `material_id` 从素材源下载完整视频文件
   - 下载信息来源：`material_list..csv`

3. **提取场景片段**
   - 根据场景的起始时间和结束时间，从原始素材中截取对应片段
   - 格式：视频文件（MP4等）

4. **去除BGM**
   - 对每个场景片段进行音频处理，去除背景音乐
   - 保留人声/对白

5. **拼接新素材**
   - 按方案中的场景顺序，将处理后的片段拼接成完整视频
   - 输出N个新的素材文件

### 输出
- **新素材文件**：
  - 命名格式：`remix_material_{scheme_id}.mp4`
  - 存储路径：`./output/remix_material_{scheme_id}.mp4`
- **处理日志**：
  - 记录每个素材的来源、处理时间、文件大小等
  - 文件：`processing_log.json`

### 技术要求
- 视频处理：使用 `ffmpeg` 等工具进行视频截取和拼接
- 音频处理：BGM分离和去除
- 文件管理：自动创建输出目录，管理临时文件
